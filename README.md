## 📖 What is AgentOS?

AgentOS is a dynamic AI Organization Operating System. It allows anyone—whether a founder, product manager, or developer—to describe a company, software task, or business goal in plain language. From that simple description, AgentOS automatically generates a hierarchy of specialized AI agents (like a CEO, CTO, Product Manager, and Lead Developer) that collaborate in real-time to execute that exact goal. 

Unlike massive developer frameworks that require complex setups or expensive enterprise agent environments, AgentOS fills the consumer gap: it provides an elegant UI, a lightweight backend, and zero-code requirements to orchestrate a sophisticated AI workforce.

---

## 🚀 Key Features

*   **Dynamic Role Generation**: The system dynamically infers the necessary agent roles based on your plain-text project description. If you ask for a Fintech app, you get a Legal Counsel agent. If you ask for a Social Media app, you get a Marketing Director.
*   **Parallel DAG Execution (Layered AI)**: Agents don't just talk randomly. They are organized into a Directed Acyclic Graph (DAG). Layer 0 agents (Independent Strategists like the CEO) execute first. Once they finalize the high-level strategy, Layer 1 agents (Executors like Developers) begin their tasks, strictly depending on the context generated by Layer 0.
*   **Inter-Agent Message Bus**: Agents automatically exchange inquiries. If the Developer needs to know the database constraints, they can query the CTO.
*   **Real-Time Token Streaming**: Watch the AI agents "think" and type in real-time. Live outputs are streamed character-by-character to the frontend via WebSockets.
*   **Interactive SVG Visualizer**: The frontend renders a dynamic, animated organizational chart connecting the active agents, displaying their live statuses (Pending, Running, Waiting, Done, Error).
*   **Session Persistence & Exporting**: All execution history, agent configurations, and inter-agent messages are persisted in a local SQLite database. Completed sessions generate a downloadable Business Blueprint (JSON/Markdown).

## 🧠 System Architecture

AgentOS is built with a lightweight yet powerful stack optimized for rapid asynchronous execution.

```mermaid
graph TD;
    Client[Browser Frontend (HTML/JS)] <-->|HTTP REST| API[FastAPI Server]
    Client <-->|WebSocket Stream| WS[WebSocket Manager]
    API --> DAG[DAG Async Execution Engine]
    DAG --> LLM[Gemini API / LLM Providers]
    DAG <--> DB[(SQLite Database)]
    API <--> DB
```

### 1. The Database Schema (SQLite)
AgentOS persists state locally via `agentos.db`.
- **`sessions`**: Represents a single project execution. Stores the user's initial prompt (`description`), status (draft/running/completed), and timestamps.
- **`agents`**: The spawned AI entities. Each agent belongs to a session, has a specific `role` (e.g., CEO, CTO), assigned LLM `model`, `system_prompt`, `task`, and belongs to a specific dependency `layer`.
- **`messages`**: Acts as the inter-agent communication bus. Records direct messages between `from_agent` and `to_agent` to simulate realistic team collaboration.
- **`results`**: Caches the final consolidated outputs, syntheses, and key metrics extracted from the agent collaboration.

### 2. The Backend Engine (FastAPI & Python 3.11+)
- **Dynamic Generator**: When a session is created, the backend asks the LLM to act as an HR Manager, inferring exactly which roles are needed for the specific prompt, writing customized `system_prompts` for each role.
- **The DAG Executor (`execute_dag`)**: This is the heart of AgentOS. It uses Python's `asyncio` to group agents by their `layer` property. It schedules `asyncio.gather` for all Layer 0 agents, waits for their completion, aggregates their output into a shared context, and then schedules Layer 1 agents.
- **WebSocket Manager**: Provides the real-time token streaming (`/ws/sessions/{id}`). Background tasks yield LLM tokens and broadcast them via `ConnectionManager.broadcast()` pushing live JSON to clients.

### 3. The Frontend Client (Vanilla ES6)
We opted for zero heavy JS frameworks (No React/Vue) to ensure maximum speed and minimal bundle sizes.
- **`index.html`**: The prompt composer.
- **`session.html`**: The execution dashboard. Connects to the WebSocket, listens for `AGENT_STATUS` updates, and prints live text via `ws.js`.
- **`orgChart.js`**: Dynamically calculates DOM element positions and draws SVG lines (`<path d="..." />`) connecting Layer 0 to Layer 1 agents, animating the DAG dependencies.
- **`agentPanel.js`**: The slide-out inspector allowing users to manually override an agent's assigned LLM model mid-flight.

## 🏁 Getting Started (Local Setup)

### 1. Prerequisites
Ensure you have the following installed:
*   [Python 3.11 or higher](https://www.python.org/downloads/)
*   An API Key from [Google AI Studio](https://aistudio.google.com/) (Free tier available)

### 2. Clone & Environment
```bash
git clone https://github.com/mrigeshkoyande/AgentOS.git
cd AgentOS
```

Create a `.env` file in the root of the project:
```env
GEMINI_API_KEY=your_gemini_api_key_here
```

### 3. Start the Backend
```bash
cd backend
python -m venv venv
# Windows: .\venv\Scripts\activate
# Mac/Linux: source venv/bin/activate
pip install -r requirements.txt

python main.py
```
*The FastAPI server will start at `http://localhost:8000`.*

### 4. Start the Frontend
In a new terminal window, serve the static frontend folder:
```bash
python -m http.server 3000 --directory frontend
```
*Open `http://localhost:3000` in your browser!*

## 📡 REST API Reference

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/api/sessions` | Create a new session and dynamically spawn agents based on the description. |
| `GET` | `/api/sessions` | Fetch all historical sessions. |
| `GET` | `/api/sessions/{id}` | Load details of a specific session (including its agents list). |
| `POST` | `/api/sessions/{id}/run` | Start the asynchronous DAG execution engine. |
| `PATCH` | `/api/agents/{id}/model` | Override the assigned LLM model for a specific agent. |
| `GET` | `/api/sessions/{id}/results` | Fetch the final generated business blueprint/report. |
| `GET` | `/api/sessions/{id}/export` | Export the report as downloadable `.json` or `.md`. |
| `DELETE`| `/api/sessions/{id}` | Remove a session and its associated data from DB. |
| `WS` | `/ws/sessions/{id}` | WebSocket route for live agent status and tokens broadcasts. |
